# Contributing to Sturna.ai

Thank you for your interest in contributing to Sturna.ai!

## Code of Conduct

This project follows a professional and respectful code of conduct. Please be kind and constructive in all interactions.

## How to Contribute

### Reporting Issues
- Use the GitHub Issues tab.
- Provide clear reproduction steps, expected behavior, and environment details.
- For security issues, please email security@sturna.ai instead of opening a public issue.

### Pull Requests
1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes.
4. Ensure your code follows the existing style and includes tests where applicable.
5. Submit a pull request with a clear description of the changes.

## Development Setup

```bash
git clone https://github.com/stephentrembley-oss/Sturna.ai.git
cd Sturna.ai
pip install -r requirements.txt
```

For the Octomind (Node.js) layer:
```bash
cd octomind
npm install
```

## Commit Guidelines
- Use clear, descriptive commit messages.
- Reference issues where relevant (e.g., `fix: resolve agent timeout (#123)`).

## Questions?

Open an issue or reach out via the project discussions.

Thank you for helping improve Sturna.ai!
